# AI 讲义生成规范

本规范供 Codex、Claude Code、ChatGPT、Gemini 等 AI 工具在本项目中直接加载使用。目标不是做摘要，而是把课件、题目、表格、图示重构成一份闭环、可打印、可复习、可替代原材料的 HTML 讲义。

推荐与 `docs/codex_closed_loop_lecture_workfile.md` 配套使用：前者定义规则，后者定义这一次任务的输入、输出和执行顺序。

## 1. 总原则

1. 以“闭环重构”为目标，不做松散摘要
2. 以中文为主，关键术语首次出现时补简洁英文
3. 所有输出必须自洽，不能把理解责任推回原文件
4. 若源材料存在跳步，可做必要前置补全，但必须明确标注
5. 若发现 API 可用，也只能作为可选增强，不能让离线主流程失效

## 2. 必须覆盖的内容

输出讲义时，必须完整迁移并解释以下内容，只要源材料中出现就不能无故略去：

- 概念
- 定义
- 公式
- 定理
- 图表
- 例题
- 推导
- 比较
- 结论
- 题目与解答

禁止出现下列表达：

- 见原文
- 略
- 自行阅读
- see original
- any wording that pushes understanding back to the source files

## 3. 补全规则

允许补全，但必须同时满足：

- 仅补理解当前内容所必须的前置知识
- 明确标注为“必要前置补全”
- 不喧宾夺主，不扩成与用户文件无关的百科条目
- 不臆造原文件不存在的实验数据、结论或题目答案

## 4. 语言规则

- 主体使用中文
- 关键术语首次出现时写成“中文（English）”
- 不要逐行双语翻译
- 不要输出一个纯中文、没有英文术语锚点的版本

## 5. HTML 输出规则

主产物必须满足：

- 恰好是一份完整 HTML
- 放在一个代码块里
- 可以直接另存为 `.html`
- 内嵌 CSS
- 支持打印
- 数学公式使用 LaTeX
- 可使用 MathJax CDN
- 不要使用浮动目录、浮动按钮、sticky 顶栏等会破坏打印体验的元素

## 6. HTML 结构要求

输出的 HTML 主体必须包含以下板块，且顺序建议保持稳定：

1. 10 分钟速记区
2. 全理论闭环重构
3. 必要前置补全
4. 图示重构
5. 绿色纸笔练习区
6. 全部例题 / 题目完整解答

### 6.1 图示重构要求

- 优先将结构图、流程图、关系图改写为内联 SVG
- 如果原图无法完整恢复，也要用结构化文字说明图中对象与关系
- 不能简单写“图略”或只贴一句图片标题

### 6.2 绿色纸笔练习区要求

- 用浅绿色背景块突出
- 提供可手写推导、概念复述或小题练习的空间
- 练习题应紧扣本节内容，不应脱题扩散

## 7. 数学规则

- 所有公式都必须解释变量含义
- 源材料给出推导时，要逐步重构
- 如果只给结论但理解依赖推导，可做狭义必要补全
- 不要只罗列结论而省略关键逻辑链

## 8. 自检输出要求

在 HTML 代码块之外，必须附一段“自检 / 覆盖核对”，至少说明：

- 是否覆盖了全部 slide 或全部章节
- 是否补写了必要前置补全
- 是否处理了图示与例题
- 是否仍存在缺失材料、不可恢复图示或待人工确认点

“自检 / 覆盖核对”不能放进 HTML 主体中。

## 9. API 模式约束

- 若检测到 `.env` 中存在 API Key，可提示用户是否启用 LLM 模式
- LLM 模式是可选增强，不是前置条件
- 使用外部 API 可能产生费用
- 处理敏感材料前，应提醒用户注意隐私与合规

## 10. 推荐输出流程

1. 先读取 PPT 提取得到的 IR、QA 报告和补充材料
2. 列出本节必须覆盖的知识点
3. 判断哪些内容需要“必要前置补全”
4. 重建理论结构、图示、题目与解答
5. 最后输出完整 HTML 和外部自检说明

## 11. 最小提示模板

你不是摘要器，而是学术讲义重构器。请基于输入材料，输出一份完整、闭环、可打印的 HTML 讲义：

- 必须覆盖全部核心概念、公式、图示、例题与解答
- 必须包含 10 分钟速记区、全理论闭环重构、必要前置补全、图示重构、绿色纸笔练习区、全部例题 / 题目完整解答
- 必须使用中文主述、关键术语首次附英文
- 必须在 HTML 代码块外附带自检 / 覆盖核对
- 不得使用“见原文”“略”“自行阅读”等表达
