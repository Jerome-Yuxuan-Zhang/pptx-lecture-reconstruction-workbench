# 无损课程材料编译规范

本规范用于把课程材料先编译成“无损底稿”，再进入闭环讲义重构。它不是摘要规范，而是**原文保留 + 结构化重组 + 全量审计**规范。

推荐把它与以下文件配套使用：

- `docs/codex_lossless_compilation_workfile.md`
- `prompts/lossless_course_material_compiler_prompt.md`
- `docs/lecture_note_generation_spec.md`

建议工作流固定为两阶段：

1. 先执行“无损课程材料编译”
2. 再基于无损底稿执行“闭环讲义重构”

## 1. 目标定义

AI 在本规范下不是总结器，不是提炼器，而是“课程材料无损整理引擎”。

它的唯一任务是把输入材料整理成：

- 结构化
- 可检索
- 可追溯
- 可审计
- 可续传
- 尽量无删减

## 2. 核心约束

### 2.1 Preservation First

1. 不得删减任何信息。
2. 不得省略定义、例子、注释、图表信息、脚注、老师提醒、题目、公式、术语、边界条件、限定语。
3. 不得把多条信息压缩成一句泛化表述。
4. 不得把“整理”做成“总结”。
5. 允许重组顺序，但不允许丢失内容。
6. 若原文存在重复，重复也要保留，并显式标记“原文重复出现”。
7. 若存在 OCR 错误、截图残缺、图像模糊、公式不完整，必须标记，不得擅自补写。
8. 不得使用外部知识修补课程内容，除非用户明确要求。

### 2.2 Reorganization Without Compression

允许：

- 增加来源标记
- 增加检索标签
- 增加风险标记
- 按主题重组
- 交叉索引

不允许：

- 合并定义
- 抽重点替代原文
- 用一句话代替一整段
- 因“冗余”而删除重复

### 2.3 Separation of Layers

必须严格区分三层：

- 原文层：只保留原材料内容，不做扩展
- 整理层：只做结构调整、索引和映射，不做压缩
- 说明层：解释、教学展开、补充说明

默认只输出“原文层 + 整理层”，不要主动进入说明层。

## 3. 固定输出结构

收到材料后，输出顺序必须固定为以下 8 个部分。

### 3.1 Source Register

记录：

- 材料编号
- 文件名或用户给定名称
- 章节 / 周次 / Topic
- 页码 / 图片序号 / 段落编号 / 时间戳
- 材料类型
- 本次输入范围

### 3.2 Lossless Transcription

要求：

- 保留原有标题、层级、编号、项目符号
- 保留术语、公式、数字、括号、限定语
- 保留例子、反例、备注、口头提醒
- 表格尽量重建；无法重建时逐行逐列展开
- 图示无法直接嵌入时，逐元素描述可见文字、箭头、坐标轴、图例、标签与关系
- 公式必须原样保留，不能只写解释

### 3.3 Structured Reorganization

在不减信息量的前提下，将内容重组到以下类别：

- 概念与定义
- 机制 / 原理 / 流程
- 模型 / 框架 / 分类
- 例子 / 案例 / 应用
- 限定条件 / 例外 / 易错点
- 公式 / 图表 / 数据
- 老师强调 / 口头补充 / 考点提示
- 习题 / 问题 / 答案 / 线索

每一条都应尽量保留原始位置引用。

### 3.4 Concept Index

为每一个概念、术语、缩写、模型名、理论名、变量名建立索引。每项应包含：

- 术语原文
- 中文 / 英文对应
- 出现位置
- 原文定义
- 相关上下文
- 与其他术语的关系
- 是否存在近义混淆
- 是否属于默认前置知识

### 3.5 Dependency Map

只根据材料本身识别：

- 材料中已解释的前置知识
- 材料默认读者已会、但未展开的前置知识
- 前置缺口列表

不得擅自补讲，只能识别和标记。

### 3.6 Ambiguity & Risk Log

必须记录：

- OCR 错误
- 图像模糊
- 公式残缺
- 缺页
- 指代不明
- 术语未定义直接使用
- 同概念多种说法
- 口头表达不严谨
- 上下文缺失

每条都应写：

- 风险点
- 所在位置
- 为什么有风险
- 是否需要补材料

### 3.7 No-loss Audit Checklist

必须逐项确认是否保留了：

- 标题
- 小标题
- 编号
- 项目符号
- 所有定义
- 所有例子
- 所有反例
- 所有公式
- 所有图表信息
- 所有表格信息
- 所有数字
- 所有限定语
- 所有转折语
- 所有脚注 / 备注
- 所有老师提醒
- 所有题目
- 所有答案线索
- 所有重复内容
- 所有不确定内容标记

若任何一项无法确认，必须明确写“未能确认完整保留”。

### 3.8 Continuation Protocol

若单轮无法处理完，必须明确输出：

- 已处理范围
- 未处理范围
- 下一段起点
- 需要延续的实体 / 术语 / 编号
- 尚未解决的歧义

## 4. 强制禁止事项

1. 严禁输出“精简版总结”。
2. 严禁用“主要内容如下”替代全量展开。
3. 严禁把多个定义合并成自己的概括。
4. 严禁把图表概括成一句话。
5. 严禁因为“看起来不重要”而省略老师补充。
6. 严禁为了排版整齐而删除重复内容。
7. 严禁用外部常识修复原文漏洞。
8. 严禁在未完成审计前声称“整理完成”。

## 5. 与本项目的集成方式

推荐输入：

- 原始 PPT / PDF / 讲义 / 课堂截图 / 录音转写
- `slide_ir.json`
- `qa_report.md`
- 图形降级资产 `assets/`

推荐输出：

- `artifacts/material_compiler/<batch>/lossless_register.md`
- `artifacts/material_compiler/<batch>/lossless_compilation.md`
- `artifacts/material_compiler/<batch>/ambiguity_risk_log.md`
- `artifacts/material_compiler/<batch>/continuation_state.md`

## 6. 与闭环讲义规范的关系

本规范优先解决“绝不先丢信息”的问题。

`docs/lecture_note_generation_spec.md` 优先解决“如何把材料重构为闭环讲义”的问题。

两者不是替代关系，而是上下游关系：

- 上游：无损课程材料编译
- 下游：闭环讲义重构

## 7. 一句话指令

当用户提供课程材料时，AI 应直接开始：

1. 建立 Source Register
2. 执行 Lossless Transcription
3. 执行 Structured Reorganization
4. 建立 Concept Index
5. 输出 Dependency Map
6. 输出 Ambiguity & Risk Log
7. 输出 No-loss Audit Checklist
8. 如需续传，输出 Continuation Protocol
